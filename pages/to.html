Ternary Operator ?:

назад

Тернарный оператор в JavaScript — это компактный условный оператор, позволяющий написать if ... else в виде выражения. Его синтаксис:

  
    condition ? exprIfTrue : exprIfFalse

Где condition — выражение (приводится к boolean), exprIfTrue и exprIfFalse — любые выражения. Тернарный оператор возвращает значение — в отличие от if/else, это выражение, а не инструкция. Поэтому его можно использовать там, где ожидается значение (присвоение, возврат из функции, шаблонная строка, JSX и т.д.).

Ключевые свойства и важные моменты

  • Это выражение, а не инструкция: let x = a ? b : c;.
  • Право-ассоциативен: a ? b : c ? d : e читается как a ? b : (c ? d : e).
  • Выполняется (оценится) только та ветвь, которая выбрана — вторая ветвь не вычисляется (как if).
  • Оба результата могут быть любого типа — числа, строки, объекты, функции и т.д.
  • Нельзя опускать : exprIfFalse — обе части обязательны.
  • Хорош для короткой логики, но злоупотребление (особенно вложенные тернарники) ухудшает читаемость.

Простые примеры

Базовый пример:

  
    const age = 20;
    const canDrink = age >= 21 ? 'yes' : 'no';
    console.log(canDrink); // 'no' (в большинстве стран порога 21)

Использование в присвоении:

  
    let fee = isMember ? 2.0 : 10.0;

Вызов функции в ветвях:

  
    const result = isOk ? doWhenOk() : doWhenFail();

В шаблонной строке:

  
    const user = { name: 'Игорь', isAdmin: false };
    const label = `${user.name} — ${user.isAdmin ? 'админ' : 'пользователь'}`;

Условный className:

  
    const isActive = true;
    const className = `btn ${isActive ? 'btn--active' : 'btn--idle'}`;
    // btn btn--active

JSX (React):

  
    // рендерим разный контент в зависимости от состояния
    function Welcome({ user }) {
      return (
        <div>
          {user ? <p>Привет, {user.name}!</p> : <p>Пожалуйста, войдите.</p>}
        </div>
      );
    }

    // короткий вариант для небольших ветвлений (но осторожно с читабельностью)
    <div>{isLoading ? 'Загрузка...' : <AppContent />}</div>

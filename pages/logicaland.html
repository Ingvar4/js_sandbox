Logical AND

назад

&& — это логический оператор И (AND). Он:

  • коротко-замыкающий (short-circuiting): если левый операнд приводится к false (т.н. falsy), правый операнд не вычисляется и возвращается значение левого;
  • не обязательно возвращает булево — в JavaScript && возвращает один из своих операндов (первый falsy или последний truthy), а не обязательно true/false.

Примеры

  
    // 1. Простая логика
    true && false        // => false  (в булевом контексте)
    true && "hello"      // => "hello"  (true -> возвращаем правый)
    0 && doSomething()   // => 0, doSomething() не вызовется (0 falsy)

    // 2. Возврат первого falsy или последнего truthy
    "foo" && 42 && {}    // => {}   (все truthy, возвращаем последний)
    "foo" && "" && 42    // => ""   ("" — первое falsy, возвращается)

    // 3. Guard / защита перед доступом к свойству (раньше часто использовали)
    const user = null;
    const name = user && user.name; // => null (без ошибки), но лучше user?.name

    // 4. Условный вызов функции (практичный паттерн)
    const maybeFn = null;
    maybeFn && maybeFn(); // ничего не произойдёт, т.к. maybeFn falsy

    // 5. Условный рендер в React JSX
    // {condition && <Component />} — если condition == 0, в DOM попадёт 0, будьте осторожны
